Chapter 1: What is a Computer? Hardware vs Software

Definition of a Computer

A computer is an electronic device designed to process data. It takes in information, works on it, and then gives us a result.

Every single action a computer performs, from opening a web page to playing a video, follows the IPSO cycle.

The IPSO Model

Input, Process, Storage, Output - This model applies to everything from a basic calculator to the most powerful supercomputer.

  • Input: How we feed data or instructions into the computer (keyboards, mice, microphones, touchscreens)
  • Process: The computer's brain, the CPU, performs calculations and manipulates data based on instructions
  • Storage: Where the computer keeps data and programs, both temporarily (RAM) and permanently (HDD/SSD)
  • Output: How the computer presents results back to us (screens, speakers, printers, or controlling other devices)

The Three Layers

Layer Description Examples
Hardware Everything physical about a computer - anything you can literally touch CPU, RAM, Motherboard, Keyboard, Monitor
Software Set of instructions or programs that tells the hardware what to do (intangible) Windows, macOS, Chrome, Word
Firmware Special software permanently programmed into hardware - the bridge between them BIOS/UEFI

Software Types

System Software

Manages the computer resources

Examples: Operating Systems (Windows, macOS, Android)

Application Software

Performs specific tasks for the user

Examples: Microsoft Word, Google Chrome, programs you use to get things done

How They Interact - The Layered System

  1. At the top is you, the User
  2. You interact with an Application
  3. That application sends a request to the Operating System
  4. Which then uses firmware and drivers
  5. To give low-level instructions to the Hardware

This layered approach makes computing efficient and organized.

Key Point: These three layers are completely codependent. Hardware is just metal and plastic without software. Software is just code without hardware to run on. Firmware is the essential glue that lets them talk to each other.

Chapter 2: Motherboards - Evolution & Functions

What is a Motherboard?

The motherboard is the main circuit board that every hardware and physical component is plugged to. If the CPU is the brain, the motherboard is the nervous system, carrying signals and instructions between every single part.

Also known as: Mainboard, Logic Board (in Apple products)

Without it, the components would be isolated and useless.

Key Functions of the Motherboard

Function Description
1. Power Management/Distribution Takes power from the PSU and ensures every component (CPU, RAM, etc.) gets stable electricity with correct voltage
2. Communication Center A communication hub using a network of electrical pathways called "buses" (data highways) to let components talk to each other
3. Connectivity Provides ports on the back for external devices/peripherals (USB, Ethernet, Audio)
4. Expansion Offers slots to add or upgrade components like graphics cards (PCIe slots) and storage

Major Components on the Motherboard

  • CPU Socket: Large square where the processor fits (must match your CPU brand!)
  • RAM Slots: For memory sticks (short-term memory)
  • Chipset: Controls data flow - often under a heatsink (Northbridge + Southbridge or modern PCH)
  • PCIe Slots: Long slots for GPU and other expansion cards
  • Storage Connectors: SATA and M.2/NVMe for hard drives and SSDs
  • Power Connectors: 24-pin and 8-pin to provide electricity
  • Rear I/O Ports: USB, Ethernet, audio, video outputs
  • BIOS/UEFI Chip: Stores firmware
  • CMOS Battery: Coin battery that keeps BIOS settings saved

Motherboard Form Factors (Sizes)

Form Factor Description Use Case
ATX Full-sized, most expansion slots Standard desktops, gaming PCs
Micro-ATX Smaller, fewer slots Compact builds
Mini-ITX Very small, minimal slots Tiny PCs, HTPCs

Trade-off: Size vs. Expandability. Choose based on your case size and upgrade needs.

Chipsets: Communication Hub

Old Design (Two-Chip):

  • Northbridge: Fast communication (CPU, RAM, GPU)
  • Southbridge: Slower tasks (USB, SATA, PCI)

Modern Design:

  • CPU: Contains Northbridge functions built-in
  • PCH (Platform Controller Hub): Single chip manages everything else

This modern design is much faster and more efficient!

Buying Tip - CRITICAL: CPU socket MUST match! Intel CPUs need Intel sockets. AMD CPUs need AMD sockets. They're not interchangeable. Always check compatibility!

Chapter 3: Chipsets, BIOS/UEFI & The Boot Process

Why Power & Booting Matter

A computer is just a collection of parts until it gets two things:

  1. Stable Power: The right voltage for each component
  2. Instructions: A wake-up script to get everything started

This process is called booting.

BIOS vs UEFI

Feature BIOS (Old) UEFI (Modern)
Interface Text-only Graphical, Mouse support
Boot System MBR (Master Boot Record) GPT (GUID Partition Table)
Drive Limit 2.2 TB max Effectively unlimited
Speed Slower to initialize Faster (parallel initialization)
Security Basic password Secure Boot

The Boot Process - 6 Steps

  1. Power On: You press button â†’ PSU sends "power good" signal to motherboard
  2. BIOS/UEFI Awakens: CPU loads firmware from ROM chip
  3. POST (Power-On Self-Test): BIOS checks critical hardware (CPU, RAM, GPU). Beep codes or LED indicators show errors
  4. Find Boot Device: BIOS checks boot priority (1. SSD, 2. USB, etc.)
  5. Load Bootloader: Finds bootloader on storage device and hands over control
  6. OS Loads: Bootloader loads Windows/Linux/macOS into RAM, OS takes control

Where Settings Are Saved

  • Firmware Program (BIOS/UEFI): Stored on SPI Flash ROM chip (non-volatile)
  • Your Settings (boot order, time): Stored in CMOS RAM (volatile - needs power)
  • Coin Battery: Powers CMOS RAM and Real-Time Clock when PC is off
If battery dies: PC still boots, but you must reset time and settings each time!

Power Supply Unit (PSU)

Before anything can boot, it needs power. The PSU converts AC power (from your wall) into stable DC power for components:

  • +12V: For power-hungry parts (CPU, GPU)
  • +5V: For USB ports and some drives
  • +3.3V: For sensitive chips and RAM

Wattage: Must be enough to power everything (e.g., 750W for gaming PC)

Chapter 4: CPU - From Logical Architecture to Physical Silicon

What is the CPU?

The CPU is the brain of the computer. Its job is to fetch, interpret, and execute instructions from software at incredible speed - billions of operations per second.

Every action you take (moving the mouse, playing music, opening a file) is processed in the CPU.

CPU Core Components

Control Unit (CU)

The manager - fetches and decodes instructions, directs data flow

ALU (Arithmetic Logic Unit)

The mathematician - does all math (add, subtract) and logic (AND, OR, NOT)

Registers

Super-fast memory inside CPU that holds data the ALU is currently working on. Each register is made of flip-flops (1 flip-flop = 1 bit).

Important Registers

Register Name Function
PC Program Counter Stores address of NEXT instruction
IR Instruction Register Holds CURRENT instruction being executed
AC Accumulator Stores ALU operation results
MAR Memory Address Register Holds memory address to access
MDR Memory Data Register Holds data being read/written to memory

Instruction Cycle - The CPU's Heartbeat

  1. Fetch: CU fetches next instruction from memory (using PC)
  2. Decode: CU decodes the instruction (in IR)
  3. Execute: CU sends operation to ALU
  4. Store: Result written to register (AC) or back to memory
This cycle repeats BILLIONS of times per second!

Clock Speed

Clock Cycle: The fundamental "tick" of the CPU

Clock Speed (GHz): Number of cycles per second

  • 1 GHz = 1 billion cycles/second
  • 4 GHz CPU = 4 billion ticks/second
  • One cycle time = 1 / 4,000,000,000 = 0.25 nanoseconds!
Clock Cycle Time = 1 / Clock Speed

CPI (Cycles Per Instruction)

Definition: Average number of clock cycles needed to complete ONE instruction

  • Simple instruction (ADD): 1 cycle
  • Complex instruction (MULTIPLY): 3 cycles
  • Load from RAM: 100+ cycles!

Goal: Lower CPI = more efficient CPU

How to Reduce CPI (Make CPU Faster)

  • Pipelining: Overlap execution of multiple instructions
  • Superscalar: Multiple execution units work in parallel
  • Out-of-Order Execution: Run instructions in most efficient order
  • Branch Prediction: Predict program flow to avoid stalls
  • Large Cache: Reduce slow RAM access

Chapter 5: Memory - RAM, ROM, Cache, Storage Hierarchy

Why Does Memory Matter?

The CPU is the brain, but a brain with no memory is useless. We need:

  • A place to work - short-term memory (RAM)
  • A place to store things for later - long-term memory (HDD/SSD)

Bottleneck Analogy: Imagine a chef (CPU) who can chop 100 vegetables/minute. If you (memory) can only hand him 1 vegetable every 10 seconds, it doesn't matter how fast he is. That slow hand-off is the bottleneck.

Bottleneck: The component that limits overall performance. Your computer is only as fast as its slowest part.

The Memory Hierarchy

Why don't we just use one giant, super-fast memory? Because it would be impossibly expensive.

Level Location Speed Size Cost per GB
1. CPU Registers Inside CPU Fastest Bytes Most expensive
2. Cache (L1, L2, L3) On CPU chip Extremely fast KB to MB Very expensive
3. RAM On motherboard Fast GB (4-64GB) Moderate
4. SSD/HDD Connected storage Slower TB (100s of GB+) Cheapest/GB

The system works by moving data between these levels efficiently.

Cache Memory

Cache is the CPU's personal assistant. It's tiny, super-fast memory built right onto the CPU chip. It predicts what data the CPU will need next and grabs it from RAM before the CPU even asks.

  • L1 Cache: Tiny (KB), fastest, separate Instruction & Data caches
  • L2 Cache: Larger (KB-MB), slightly slower
  • L3 Cache: Largest (MB), slowest (still incredibly fast), shared by all CPU cores

You don't buy cache separately - it's part of the CPU. When buying a CPU, L3 cache size is a key specification.

RAM vs ROM

Feature RAM (Random Access Memory) ROM (Read-Only Memory)
Volatility Volatile (loses data when power off) Non-volatile (keeps data)
Purpose Working memory for OS & apps Stores BIOS/UEFI firmware
Can Write? Yes (Read & Write) Read-only (or rarely written)
Speed Fast Slower

Key Point: When you double-click an icon, the program moves from slow storage to fast RAM so the CPU can work on it. You must SAVE your work to move it from volatile RAM to permanent storage!

How to Choose RAM

  1. Form Factor: DIMM (desktops) or SO-DIMM (laptops)
  2. Generation: DDR3, DDR4, or DDR5 - MUST match motherboard!
  3. Capacity:
    • 8GB: Bare minimum
    • 16GB: Sweet spot for most users
    • 32GB: Heavy multitasking, developers
  4. Speed: 3200MHz, 6000MHz - faster is better if motherboard supports
  5. Configuration: Buy in pairs (2x8GB better than 1x16GB) for dual-channel bandwidth doubling

HDD vs SSD

Feature HDD (Hard Disk Drive) SSD (Solid State Drive)
Technology Spinning magnetic platters Flash memory chips (NAND)
Speed 50-150 MB/s 500-7000+ MB/s
Boot Time 40-90 seconds 7-15 seconds
Durability Fragile (moving parts) Shock resistant (no moving parts)
Noise Whirring/clicking 100% silent
Cost Very cheap More expensive per GB
Best For Bulk storage, backups OS, applications, speed
RULE: Use an SSD for your Windows/OS and apps. Use an HDD only for storing photos, videos, and backups.

Upgrade Decision Guide

Upgrade to SSD if:

  • PC is slow to boot
  • Apps take 30+ seconds to load
  • You hear clicking/spinning
  • Task Manager shows 100% disk usage

Result: Biggest speed improvement! Boot: 60s â†’ 10s

Upgrade RAM if:

  • PC boots fast but lags with many tabs
  • Slow when running multiple apps
  • Task Manager shows 90%+ RAM usage

Result: Better multitasking, can run more programs

Flash Memory Types (Inside SSDs)

Type Bits/Cell Speed Durability Use Case
SLC 1 Fastest 100K cycles Enterprise servers
MLC 2 Fast 10K cycles High-end consumer
TLC 3 Standard 3K cycles Most consumer SSDs
QLC 4 Slower 1K cycles Budget drives

Note: TLC is the standard for most good SSDs today.
